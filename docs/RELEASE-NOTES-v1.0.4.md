# Smart Replay Tracker v1.0.4

## What's fixed

- Fixed an issue where Auto Remux could occasionally leave two `mp4` files for the same replay
- The plugin no longer uses a fallback file copy for replay moves, which makes it less likely to create a small partial file plus a second `_1.mp4` file

## Upgrade notes

- Replace `smart-replay-tracker.dll` with the new file from `release/`
- `replay-file-organizer.lua` does not require a change for this patch

## Notes

- OBS log lines such as `WASAPI: Device invalidated`, `window disappeared`, or `audio is lagging` are not generated by this plugin
- Those messages come from OBS audio capture sources and are outside the scope of this patch
